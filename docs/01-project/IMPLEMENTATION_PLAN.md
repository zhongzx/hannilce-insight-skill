# MBTI 话题采集系统 · 实施方案

> 状态：✅ 已完成
> 版本：v0.1（初稿，待评审后迭代）

---

## 1. 项目目标

**一句话：** 通过多轮开放式话题对话，让 AI 在不直接提问 MBTI 的前提下，推断用户的性格类型。

**核心原则：**
- 不问"你是内向还是外向"，而是通过话题设计让答案自然浮现
- 不干预用户的回答方向，保真优先
- 质量下降时封存话题，不追问，避免引导失真

## 3. 核心模块设计

### 3.1 用户标识

| 设计项 | 方案 |
|--------|------|
| 触发方式 | `/MBTI 姓名` |
| 唯一标识 | 姓名 + 首次认证时间戳 → SHA256 前8位 |
| 唤醒机制 | 新开 Session，LLM 压缩前情至 system prompt 继承上下文 |

### 3.2 话题生成

```
新闻采集（4源定时抓取）
      │
      ├── 成功 → 随机选取一条作为种子话题
      │
      └── 失败（24h内无更新）→ 内置话题池兜底（10条，覆盖四维度）
                                 ↓
                          LLM 加工话题
                          （扩展为开放式对话引子）
```

**新闻采集源（已验证可用）：**

| 源 | 状态 | 话题数/次 |
|----|------|----------|
| 虎嗅 | ✅ | ~20条 |
| 36氪 | ✅ | ~23条 |
| 今日头条 | ✅ | ~9条 |
| 财新 | ✅ | ~22条 |

**内置话题池：** 10条，覆盖 EI/SN/TF/JP 四个维度，交错排列避免连续问同一维度。

### 3.3 质量控制

| 维度 | 指标 | 说明 |
|------|------|------|
| Token层 | 回复长度 | 过短说明没展开，过长可能是背诵 |
| 语义层 | 回答相关性 | 与话题的相关程度 |
| 滑动窗口 | 连续 N 轮趋势 | 监控质量是否持续下降 |

**封存策略（质量下降不干预）：**

| 下降严重程度 | 触发条件 | 动作 |
|-------------|---------|------|
| 严重 | `decline_delta >= 0.3`，连续 2 轮 | 立即封存 |
| 中度 | `0.15 <= decline_delta < 0.3`，连续 3-4 轮 | 封存 |
| 轻微 | `decline_delta < 0.15`，连续 5 轮 | 封存 |

### 3.4 置信度评估

作为独立模块，迭代优化。初版采用三因子模型：

```
置信度 = f(Token量, 质量分, 一致性)
```

**一致性来源：**
- 同一维度的历史打分分布
- 跨维度的回答风格一致性

### 3.5 结束条件

**正常结束（同时满足）：**
- 数据足够 + 质量达标
- 四维度（EI / SN / TF / JP）全部覆盖

**提前结束（数据不足时主动退出）：**
- LLM 生成委婉拒绝语
- 不固定文案，由 LLM 动态生成

### 3.6 报告输出

| 字段 | 说明 |
|------|------|
| MBTI 类型 | 4字母，如 ENFP |
| 雷达图 | 5维度打分（待定） |
| 置信度 | 0-1 |
| 维度覆盖 | EI/SN/TF/JP 各维度覆盖情况 |
| 已用话题 | JSON 列表 |
