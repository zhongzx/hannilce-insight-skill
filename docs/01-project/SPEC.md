## 1. 项目背景

MBTI（Myers-Briggs Type Indicator）是全球最广泛使用的人格类型评估工具之一。然而传统 MBTI 测试依赖选择题问卷，存在以下问题：

- **回答失真**：用户倾向于选择「社会期望」的答案，而非真实想法
- **自我认知偏差**：用户对自身的判断往往不准确
- **缺乏过程数据**：问卷只能给出结论，无法追溯推断过程

**本项目的核心目标：** 用对话取代问卷，让 AI 通过自然聊天推断 MBTI 类型。

---

## 2. 核心价值

| 价值点 | 说明 |
|--------|------|
| **保真** | 不直接问 MBTI 相关问题，通过话题引导让答案自然浮现 |
| **可解释** | 每轮对话都有 Token 量/质量分记录，推断过程可追溯 |
| **自适应** | 质量下降时封存话题，不追问，避免引导用户走向「正确答案」 |
| **持续优化** | 置信度模型独立迭代，积累越多数据越准确 |

## 3. 用户故事

### 场景 A：首次使用

```
用户：/MBTI 张三
AI：开始采集，请问你最近有没有看过什么让你印象深刻的新闻？
用户：[回答]
AI：[打分，继续下一轮话题]
...（多轮对话）
AI：[输出 MBTI 报告]
```

### 场景 B：二次使用（唤醒）

```
用户：/MBTI 张三
AI：[压缩前情，恢复上下文]
     上次我们聊到了职业选择方向的话题，你当时提到想换一个更稳定的工作。
     最近有没有什么新的想法？
用户：[回答]
AI：[继续对话，最终更新 MBTI 报告]
```

### 场景 C：数据不足主动结束

```
用户：/MBTI 李四
AI：[对话几轮后，质量持续下降]
AI：感觉今天状态不太对，要不改天再聊？你什么时候方便？
```

---

## 4. 非功能性需求

| 需求 | 目标 |
|------|------|
| 话题新鲜度 | 优先使用当天采集的新闻，最长不超过 24h |
| 响应速度 | 单轮对话响应 < 3 秒（含打分） |
| 数据安全 | 用户数据本地 SQLite，WAL 模式保证不丢 |
| 可移植性 | JSON 导出/导入，方便备份和迁移 |

---

## 5. 技术约束

- **Python**：主要开发语言
- **SQLite**：本地存储，WAL 模式
- **无外部 API 依赖**：除 LLM 调用外，所有逻辑本地运行
- **Git 管理**：代码和文档均在 GitHub 仓库管理
